Database Schema

PostgreSQL + PostGIS with spatial indexes

Entity Relationship Diagram
🗄️ Tables
users
id: UUID (PK)
email: VARCHAR(255)
firebase_uid: VARCHAR(128)
tier: ENUM('free','premium')
stations
station_code: VARCHAR(10) (PK)
station_name: VARCHAR(150)
division: VARCHAR(50)
zone: VARCHAR(10)
geom: GEOMETRY(Point, 4326)
tracks
id: SERIAL (PK)
osm_id: BIGINT
gauge: ENUM('broad','meter','narrow')
electrified: BOOLEAN
tracks_count: INT
geom: GEOMETRY(LineString, 4326)
spacer
trains
train_number: VARCHAR(10) (PK)
train_name: VARCHAR(150)
source_station_code: FK → stations
destination_station_code: FK → stations
runs_on_days: VARCHAR(7) '1111100'
train_telemetry_logs
id: BIGSERIAL (PK)
train_number: FK → trains (CASCADE)
last_station_code: FK → stations
next_station_code: FK → stations
delay_minutes: INT
bearing: INT (0-359)
current_location: GEOMETRY(Point, 4326)
captured_at: TIMESTAMP
🔗 Relationships
trains.source → stations.code (RESTRICT)
trains.dest → stations.code (RESTRICT)
logs.train → trains.number (CASCADE)
logs.station → stations.code (SET NULL)
⚡ Spatial Indexes (GiST)
idx_stations_geom ON stations USING GIST (geom)
idx_tracks_geom ON tracks USING GIST (geom)
idx_telemetry_geom ON logs USING GIST (current_location)
